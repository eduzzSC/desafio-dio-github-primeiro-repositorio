O que é o GitHub, afinal?
Imagine que você está escrevendo um livro. No Word, você salvaria arquivos como livro_final.doc, livro_final_v2.doc, livro_agora_vai.doc. Uma confusão, certo?

O GitHub resolve isso usando uma tecnologia chamada Git. Ele funciona como uma máquina do tempo: ele registra cada pequena alteração que você faz no seu código, permitindo que você volte no tempo se algo der errado e colabore com pessoas do mundo inteiro sem bagunçar o trabalho de ninguém.

Conceitos Chave para não se perder:
Repositório (ou "Repo"): É a pasta do seu projeto. É onde ficam guardados todos os arquivos e o histórico de mudanças dele.

Commit: Pense no commit como um "ponto de salvamento" (checkpoint). Toda vez que você termina uma tarefa, você faz um commit com uma mensagem explicando o que mudou.

Branch (Ramificação): É como uma linha do tempo alternativa. Você pode criar uma branch para testar uma ideia nova sem estragar a versão principal do projeto que está funcionando.

Pull Request (PR): É quando você termina uma alteração em uma branch e pede para o dono do projeto "puxar" suas mudanças para a versão principal.

Por que usar o GitHub?
Portfólio: É onde recrutadores olham para ver o que você sabe fazer na prática.

Backup Seguro: Seu código fica na nuvem, acessível de qualquer lugar.

Colaboração: Você pode contribuir com projetos famosos (como o VS Code ou o Linux) ou trabalhar em grupo com amigos.
